2
Agent Core — ReAct Orchestrator
Each phase is one LLM ⇄ tools conversation. Stall-detected, not call-capped — a phase ends on genuine repetition, never on an arbitrary step budget.
PHASE 1
Recon
Maps hosts, ports, services, subdomains, CVEs. Every hit is recorded live via record_target, not batched at phase end.
Nmap · crt.sh · Wayback · DNS/WHOIS
PHASE 2
Analyze
Actively probes recon results for vulnerabilities. Deterministic detectors (SQLi/XSS/redirect/RCE signals) hint the model to record a finding.
Nuclei · header/exposure audits · fingerprinting
PHASE 3
Exploit
Verifies a finding, then attempts exactly one real exploitation — only for allowlisted targets, only after a human approves it.
Metasploit · sqlmap · default-creds check
Allowlist + human approval gate
PHASE 4
Validate
Final LLM pass deduplicates overlapping findings. Every finding already carries real evidence going in — this only cleans up the set.
Deduplication only — no new tool calls

Safety & Output
Exploitation allowlist
Empty by default — populated only when an operator explicitly authorizes a target. Enforced in the tool runner itself, not by prompt convention.
Human-in-the-loop approval
The first exploitation attempt in a session pauses for one approval click in the UI; that decision then covers the rest of the session — timeboxed, and every outcome is logged.
Proof report export
One-click HTML report per session — findings, real evidence, and the full approval audit trail, ready to hand to a client.
Runs entirely locally — no cloud, no database. Session state is plain JSON on disk.
