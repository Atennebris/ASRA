Persistence
Session Store ("database")
No database server — one JSON file per session on local disk. Every target, finding, log line and approval is written the instant it happens, so a crash never loses data and the UI can poll the same file for live updates.
sessions/store.py
JSON on disk
crash-safe
Data never leaves the loop uncontrolled: Qwen only ever sees prompts/tool results the backend builds — it has no direct access to the target, the tools, or the disk.
Everything except the Qwen API call runs locally — Windows via WSL2, Linux/macOS natively.
